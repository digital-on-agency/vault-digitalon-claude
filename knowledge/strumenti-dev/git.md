# Git & Version Control

## Overview

**Git** is a distributed version control system for tracking changes in source code. Key concepts: repository, commit, branch, merge.

## Configurazioni standard

### Basic Commands

- `git clone` → copy a repo locally
- `git pull` → update from remote
- `git add` → stage changes
- `git commit` → save changes locally
- `git push` → sync with remote
- `git status` → see current state
- `git log` → see history
- `git diff` → see changes

### Working with GitHub

- Pull Requests: propose changes, review, and merge
- Repo structure and access management

### Team Workflows

- Branching strategies: main/dev/feature branches
- Merging and resolving conflicts
- Keep branches short-lived and focused

### Best Practices

- Commit often, in small chunks
- Always pull before pushing
- Communicate when merging
- Write clear commit messages
